Если был запушен "неправильный" коммит в удаленную ветку, нужно сделать git revert <хэш коммита>.
При использовании этой комманды в гите создается еще один коммит, который отменит старые изменения. Данная команда отменит лишь один коммит, если нужно отменить цепочку коммитов, то git revert <самый старый хэш коммита> <самый новый хэш коммита>.
Опять же, утилита SourceTree очень удобно показывает график веток с полной историей изменения, в т.ч. и хэши коммитов, если использовать cli, то на помощь придет git log.
Если коллеги еще не успели запулить "неправильный" коммит, можно немного схитрить и сделать git reset <хэш коммита на который нужно сделать сброс>, после этого сделать ре-коммит всех изменений и запушить ветку, тогда не останется никаких следов "неправильного" коммита.
